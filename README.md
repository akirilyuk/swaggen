# SwaggenNext

A visual platform for building APIs with OpenAPI specs, built on [Next.js](https://nextjs.org) and powered by the [swaggen](../) code generator.

## Features

- **Entity Builder** — Define data models (User, Product, Order…) with typed fields, enums, and defaults
- **Relation Editor** — Link entities with one-to-one, one-to-many, and many-to-many relationships
- **Middleware Studio** — Write custom TypeScript middlewares (auth, logging, rate-limiting) with a built-in Monaco editor
- **Service Designer** — Group operations into logical services, map HTTP methods to entities
- **OpenAPI Spec Editor** — Full Monaco JSON editor with live validation, or auto-regenerate the spec from your entities
- **Data Storage** — Built-in support for Supabase, MongoDB, MySQL, PostgreSQL, Redis, and SQLite with sample client code
- **Code Generator** — One-click generation of Next.js App Router route handlers, TypeScript interfaces, and Zod validation schemas
- **Project Management** — Create, switch between, and manage multiple API projects (persisted in localStorage)

## Getting Started

```bash
cd swaggen-next
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

Use `yarn install` / `yarn dev` if you prefer Yarn.

## Supabase database migrations

Schema changes live in `supabase/migrations/` (`001` … `007`). You can apply them **without copy-pasting SQL** into the dashboard by using the [Supabase CLI](https://supabase.com/docs/guides/cli).

### Prerequisites

- A Supabase project with `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` set in `.env.local` for the app.
- The CLI is installed as a dev dependency; use `npx supabase` or `yarn supabase`.

### One-time setup (per machine)

1. Log in so the CLI can talk to Supabase:

   ```bash
   npx supabase login
   ```

   This opens a browser and stores an access token.

2. Link this repository to your **hosted** project (do this once per clone):

   ```bash
   yarn db:link
   ```

   You will need:

   - **Project reference ID** — Supabase: **Project Settings → General → Reference ID**
   - **Database password** — the password you set when creating the project (or reset it under **Project Settings → Database**)

   This associates `supabase/migrations/` with that remote database.

### Apply migrations

Whenever you pull new migration files, push them to the remote database:

```bash
yarn db:push
```

Equivalent scripts in `package.json`:

| Script        | Command              | Purpose                                      |
|---------------|----------------------|----------------------------------------------|
| `yarn db:link` | `supabase link`     | Connect the repo to a Supabase project       |
| `yarn db:push` | `supabase db push`  | Apply pending SQL files to the remote DB     |
| `yarn db:pull` | `supabase db pull`  | Pull remote schema changes (advanced)        |

### CI / automation (optional)

For GitHub Actions or other CI:

1. Create a **personal access token** in Supabase (**Account → Access Tokens**).
2. Set the secret `SUPABASE_ACCESS_TOKEN` in your CI environment.
3. Run non-interactive `supabase link` (see [CLI reference](https://supabase.com/docs/reference/cli/introduction)) and then `supabase db push` in the pipeline so merges deploy schema changes automatically.

### Local config

`supabase/config.toml` is generated by `supabase init` and is used by the CLI. Temporary files under `supabase/.temp` are gitignored.

## Project Structure

```
src/
  app/                  — Next.js App Router pages
    api/generate/       — Server-side code generation endpoint
    api/validate/       — OpenAPI spec validation endpoint
    editor/             — Monaco-based spec editor
    entities/           — Entity builder UI
    relations/          — Relation editor UI
    middlewares/         — Middleware studio with code editor
    services/           — Service designer
    storage/            — Data storage configuration
    generate/           — Code generation + file preview
    settings/           — Project settings
  components/           — Shared UI components (Sidebar, Header, PageShell, ui primitives)
  lib/                  — Pure utilities (specBuilder, storageProviders)
  store/                — Zustand state management with localStorage persistence
  types/                — TypeScript type definitions
```

## Workflow

1. **Create a project** on the Dashboard
2. **Define entities** — your data models with fields and types
3. **Add relations** — link entities together
4. **Configure middlewares** — write business logic in TypeScript
5. **Design services** — map HTTP operations to entities
6. **Choose data storage** — pick Supabase, MongoDB, etc.
7. **Edit the spec** — fine-tune or auto-regenerate from your entities
8. **Generate code** — produce Next.js route handlers, interfaces, and Zod schemas

## Tech Stack

- **Next.js 16** with App Router
- **React 19** with Server Components
- **Tailwind CSS 4** for styling
- **Zustand** for client-side state
- **Monaco Editor** for code editing
- **Lucide React** for icons
